1 문서 사용 목적
이 문서는 압구정 S&H 듀얼 왁싱 압구정점 초회 미팅 당일에 컨설턴트가 바로 사용하는 현장 실행 프로토콜이다.
오늘 목적: 멋진 전략 발표가 아니라 구조 확정과 실행 가능성 확인이다.
  • 출발 전 준비 상태를 점검한다
  • 현장 도착 직후 확인해야 할 항목을 빠르게 통제한다
  • 미팅 60~90분을 단계별로 리딩한다
  • 기준값·사람 구조·권한 구조·채널 구조를 우선 확인한다
  • 미팅 종료 전 반드시 남겨야 할 산출물을 확보한다
  • 미팅 직후 후속 AI와 다음 회차로 바로 넘길 수 있게 정리한다
⚠ 현장에서 확인되지 않은 정보는 끝까지 미확인으로 둔다. 권한이 없으면 현장 즉시 실행으로 몰아붙이지 않고 후속 처리로 분기한다.

3 출발 전 최종 체크리스트
3-1. 장비 및 준비물
  • 노트북 본체 지참
  • 노트북 충전기 지참
  • 휴대폰 배터리 확인
  • 휴대폰 핫스팟 가능 상태 확인
  • 기준값 잠금 시트 출력 또는 오프라인 열람 준비
  • 사전 질문지 출력 또는 메모 앱 준비
  • 권한 구조 확인표 출력 또는 열람 준비
  • 현장 메모 도구 및 필기도구 준비
  • 사진 촬영 가능 장비 준비
  • 케이블/에어드롭/메신저 전송 수단 확인
3-2. 정보 및 일정 확인
  • 미팅 장소 예약/일정 재확인
  • 사전수집 주소는 참고용으로만 재확인 (확정값 아님)
  • 마스터 문서 링크 확보
  • 인터넷 불가 상황 대비 오프라인 메모 가능 상태 확인
  • 미팅 목적 문장 1줄 준비 — "오늘은 전략 확정보다 기준값·운영구조·권한구조를 정확히 잠그는 것이 목적입니다."
3-3. 동행자 역할 분담
  • 누가 질문 리딩할지 결정
  • 누가 메모할지 결정
  • 누가 채널 확인을 병행할지 결정
4 현장 도착 직후 10분 체크 포인트
4-1. 외부 확인
  • 간판이 현장 상호와 일치하는지 확인
  • 입구 유도성이 충분한지 확인
  • 층수/호수/건물 진입 동선 확인
  • 처음 방문자가 헷갈릴 요소가 있는지 확인
4-2. 현장 운영 확인
  • 현장 상호 표기 확인
  • 실제 영업 중 / 오픈 / 휴무 / 준비중 상태 확인
  • 첫 응대자 이름 또는 역할 확인
  • 첫 응대자가 실무 담당인지 확인
  • 실제 의사결정자 참석 여부 확인
  • 오늘 승인 가능한 사람 참석 여부 확인
4-3. 현장 실행 조건 확인
  • 촬영 가능 여부 확인
  • 채널 화면 확인 가능 여부 확인
  • 즉시 수정 가능한 채널 존재 여부 확인
  • 와이파이 또는 핫스팟 사용 가능 여부 확인
  • 현장 소음/대화 가능 환경 확인

5 초회 미팅 60~90분 타임라인 운영표

총 소요시간: 60~90분 범위, 현장 상황에 따라 조정

시간 대목표 컨설턴트 액션 반드시 확보할 결과
0~10분 관계 형성 및 오늘 미팅 목적 정렬 미팅 목적을 짧게 설명하고, 오늘은 기준값·역할·권한 확인이 우선임을 합의한다. 참석자 이름과 역할을 먼저 확인한다. 참석자 명단 초안, 오늘 대화 목적 합의
10~25분 기준값 잠금 시작 상호·주소·연락처·대표자·실제 운영자·신청자 여부를 질문한다. 미확인은 미확인으로 둔다. 사전수집값은 확인 질문용으로만 사용한다. 기준값 잠금 1차 메모, 확정값/미확정값 분리
25~45분 사람 구조 및 권한 구조 확인 승인권자·결재권자·실무 담당자·본사/가맹 여부·채널별 관리자 권한을 확인한다. 점포 단독 수정 가능 범위를 구분한다. 역할 구조 메모, 권한 구조 메모
45~65분 서비스/가격/채널 현황 파악 대표 서비스·가격표·예약 구조·리뷰 대응 구조·채널 현황을 확인한다. 가능하면 현장 화면 또는 자료로 교차 검증한다. 서비스 구조 메모, 가격표 확인 상태, 채널 현황 메모
65~80분 즉시 실행 가능 범위 판정 권한 있는 채널이 있는지 확인하고, 지금 가능한 일과 승인 후 가능한 일을 분리한다. 권한이 없으면 후속 요청자료로 넘긴다. 즉시 실행 목록, 승인 필요 목록
80~90분 마감 및 후속 과제 정리 오늘 확정 사실·미확인 사항·다음 요청자료·다음 회차 전 숙제를 구두로 정리하고 메모를 확인받는다. 종료 요약 5블록 초안, 다음 단계 요청자료 목록
실전 분기 원칙
로그인 가능 + 수정 권한 있음 → 현장 즉시 확인 또는 후속 반영 준비
로그인 불가 또는 권한 불명확 → 후속 처리
본사/가맹 승인 필요 → 점포 단독 실행으로 간주하지 않음
6 반드시 확인해야 할 질문 우선순위
1순위 · 기준값 잠금
  • 공식 상호를 정확히 어떻게 사용하고 계신가요?
  • 공식 주소는 무엇인가요?
  • 대표 연락처는 무엇인가요?
  • 사업자등록상 대표자는 누구인가요?
  • 오늘 기준으로 고객에게 안내하는 공식 영업 정보가 있나요?
2순위 · 사람 구조 분리
  • 실제로 매장을 운영하는 분은 누구인가요?
  • 사업 신청 또는 행정 커뮤니케이션을 맡는 분은 누구인가요?
  • 오늘 이후 가장 자주 연락할 실무 담당자는 누구인가요?
  • 현장 결정과 실행을 움직이는 핵심 인물은 누구인가요?
  • 대표자·운영자·신청자가 서로 같은 사람인가요 다른 사람인가요?
3순위 · 권한 구조 확인
  • 네이버 플레이스는 누가 관리하시나요?
  • 인스타그램은 누가 직접 운영하시나요?
  • 예약채널은 누가 관리하시나요?
  • 가격표나 소개 문구를 바로 수정할 수 있나요?
  • 비용 집행과 최종 승인은 누가 하나요?
4순위 · 서비스/가격 구조
  • 현재 핵심 서비스는 무엇인가요?
  • 대표 서비스별 가격표 최신본이 있나요?
  • 가장 많이 판매되는 서비스는?
  • 성별/부위/방식별 대표 구성은?
  • 패키지·멤버십·재방문 유도 상품이 있나요?
5순위 · 채널/리뷰
  • 현재 공식 운영 채널은 무엇인가요?
  • 각 채널 링크를 바로 보여주실 수 있나요?
  • 리뷰는 어느 채널에 가장 많이 쌓이나요?
  • 리뷰 답변은 누가 하고 있나요?
  • 예약·문의 유입은 어디서 가장 많이 오나요?
6순위 · 정책지원사업 기초 자격
  • 개인사업자인가요, 법인인가요?
  • 업력은 어느 정도인가요?
  • 직원 수는 어느 정도인가요?
  • 과거에 지원사업이나 바우처 수혜 이력이 있나요?
  • 자부담이 필요한 사업에도 검토 의향이 있나요?
채널/권한 즉시 점검 프로토콜 + 현장 메모 필수 확보 항목 컨설턴트 내부용
7 채널/권한 즉시 점검 프로토콜
채널 현장 확인 내용 권한 확인 질문 즉시 가능한 일 권한 없을 때 후속
네이버
플레이스		 채널 존재 여부, 정보 최신성, 관리자 접근 가능 여부 "플레이스는 누가 직접 관리하시나요?" 링크 확인, 정보 불일치 메모, 관리자 여부 확인 관리자 정보 요청, 권한자 연결 요청, 후속 수정 항목 목록화
인스타그램 공식 계정 여부, 최근 게시 상태, 운영 주체 "인스타 계정은 누가 운영하시나요?" 계정 확인, 최근 콘텐츠 상태 메모 운영 주체 확인 후 후속 계정 점검 요청
예약채널 사용 여부, 예약 흐름, 관리자 접근 가능 여부 "예약 채널은 누가 관리하시나요?" 플랫폼명 확인, 예약 흐름 메모 관리자 연결 요청, 접근 권 확보 후 후속 점검
카카오/
메신저		 문의 유입 여부, 응답 주체, 응답 속도 구조 "문의 답변은 누가 직접 하나요?" 메신저 운영 여부 메모 응답 담당자 지정 요청
구글
비즈니스		 보유 여부 자체가 미확인 상태 "구글 비즈니스는 등록되어 있나요?" 등록 여부 확인 메모 미등록 시 후속 검토 과제로 분리
"채널이 존재한다"와 "점포가 수정할 수 있다"를 반드시 분리해서 기록한다.

9 돌발 상황 대응 프로토콜
상황 즉시 대응 원칙 금지사항
실제 운영자 부재 오늘 참석자의 역할만 먼저 분리하고, 실제 운영자 확인 세션 또는 후속 연락 일정을 잡는다. 참석자를 실제 운영자로 단정하지 않는다.
승인권자 부재 즉시 실행안과 승인 필요 항목을 분리한다. 승인자 확인 전 수정 확정처럼 말하지 않는다. "오늘 바로 진행 가능"이라고 단정하지 않는다.
채널 로그인 불가 로그인 복구에 매몰되지 말고 관리자 주체·계정 소유권·후속 요청자료를 정리한다. 현장 시간을 비밀번호 복구에 과도하게 소모하지 않는다.
촬영 거부 촬영 없이도 구조 확인 중심으로 진행하고, 필요한 촬영은 후속 승인 항목으로 남긴다. 무리하게 촬영을 요청하거나 강행하지 않는다.
가격표 미지참 구두 확인은 임시 메모로만 남기고, 최종 가격표는 후속 자료 요청으로 분리한다. 구두 가격을 최종 확정값처럼 사용하지 않는다.
본사/가맹 구조로 즉시 수정 불가 점포 단독 실행 범위를 먼저 분리하고, 본사 승인 필요 항목을 별도 목록화한다. 독립 매장처럼 가정하고 실행안을 확정하지 않는다.
일정 지연 우선순위를 기준값 → 사람 구조 → 권한 구조 → 채널 구조 순으로 압축한다. 후순위 논의에 시간을 쓰다가 핵심 확인을 놓치지 않는다.
정책지원사업 질문 나오나 공고문 없음 후보군 수준으로만 이야기하고, 공식 공고문 확보 전 판단 보류를 명확히 한다. 지원 가능/불가·금액·선정 가능성을 단정하지 않는다.
