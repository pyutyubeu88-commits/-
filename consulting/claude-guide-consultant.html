들어가며 — 왜 대부분의 컨설턴트는 클로드를 제대로 못 쓸까요

대부분의 사용 패턴은 이렇습니다.

브라우저 열기 → "이 매장 리뷰 분석해줘" 입력 → 답변 받기 → 내일 또 처음부터.

이건 클로드를 자판기처럼 쓰는 것입니다.

예를 들어, 매번 새 클라이언트 작업을 시작할 때마다
"이 매장은 강남구 역삼동 이자카야야, 체인점이고, 사업신청자와 실제 점주가 달라, 운영 권한이 제한적이야, 리뷰 분석 기준은 강점·불만·방문목적·감성언어 4가지로 나눠줘"
라고 처음부터 설명하고 계신다면 — 그게 바로 자판기 워크플로우입니다.

진짜 업그레이드는 세 가지를 이해하는 데서 시작합니다.

주의사항

  1. Claude Design — 사용량 한도: 현재 리서치 프리뷰 단계로 주간 사용량 한도가 있습니다. 한 번에 완벽하게 만들려 하지 말고, 핵심 구조만 뽑은 뒤 Canva에서 다듬는 전략을 추천합니다.
  2. Cowork — Pro 플랜 토큰 소모: Pro 플랜에서도 사용 가능하지만 일반 채팅보다 토큰 소모가 5~20배 빠릅니다. 한도에 자주 도달한다면 Max 플랜을 고려하세요.
  3. Claude Code — npm 필요: 설치 전 Node.js(버전 18 이상)가 설치되어 있어야 합니다.
  4. 전체 공통: 모든 기능은 Anthropic 정책에 따라 변경되거나 종료될 수 있습니다. 항상 claude.ai/pricing과 support.claude.com에서 최신 정보를 확인하세요.

메모리 설정하기

경로: Settings → Capabilities
(UI 업데이트에 따라 위치가 다를 수 있습니다. Settings 안에서 찾아보세요.)

① 대화 기록으로 메모리 생성 (Generate memory from chat history)

클로드가 대화에서 맥락을 자동으로 추출해 다음 대화에도 기억합니다. 24시간마다 업데이트됩니다. 무료 플랜 포함 전 플랜에서 사용 가능합니다.

📝

예시: 오늘 "이자카야 우규 역삼점은 체인점이고 지점 자체 수정 권한은 소개 문구·사진만 가능, 사업신청자와 실제 점주가 달라"라고 말해두면 다음 주 새 대화에서도 클로드가 그 맥락을 알고 있습니다. 매번 설명하지 않아도 됩니다.

② 과거 대화 검색 (Search and reference past chats)

유료 플랜(Pro 이상)에서 사용 가능합니다. 예전 대화 내용을 자연어로 찾아줍니다.

📝

예시: "저번에 우규 역삼점 포지셔닝 어떻게 정했었지?" 하고 물어보면 클로드가 과거 대화에서 찾아서 알려줍니다.

추가 기능: 다른 AI에서 쌓은 맥락을 가져오는 메모리 가져오기 기능도 있습니다. (Settings → Memory 항목 확인)
저장 없이 대화하고 싶을 땐 우측 상단 🔒 아이콘(시크릿 모드)을 사용하세요.

2

나만의 기본 설정 작성하기

경로: Settings → Profile 또는 Settings → General

이 설정은 클로드 채팅 전체에 자동으로 적용되는 나만의 지침입니다. 한 번 써두면 매번 새로 설명하지 않아도 됩니다.

예시 — 강남구 AI 컨설턴트용 템플릿:

나는 권용준, 강남구 AI 마케팅 자립 컨설턴트입니다. 클라이언트는 주로 강남구 로컬 비즈니스 점주 (F&B·뷰티·의료·교육·피트니스). 파트너: 손미현 (기술 정비·자료 구조화 담당) 나의 컨설팅 원칙: - 설명보다 실행 — 보고서만 전달하지 않고 현장에서 바로 수정·반영 - 대행보다 자립 — 점주가 컨설팅 이후에도 직접 운영 가능한 구조를 남김 - 추상보다 현장 — 리뷰·데이터 + 입구 동선·촬영 포인트까지 직접 확인 - AI 기반 표준화 — 질문지·체크리스트·분석표·문구 템플릿으로 반복 체계화 문서 작성 규칙: - 점포명·주소를 항상 먼저 명시 (지점 혼동 방지) - 체인점이면 운영 권한 범위를 먼저 확인하고 진행 - A4 출력 기준, 체크박스·테이블 형태 선호 - 점주가 혼자서도 이어갈 수 있는 실행 단위로 마무리 현재 진행 중인 프로젝트: - 이자카야 우규 역삼점 (서울 강남구 테헤란로19길 19) - 범용 컨설팅 프레임워크 표준화 작업
⚠️

500단어 이내로 작성하세요. 너무 길면 오히려 답변 일관성이 떨어집니다.

프로젝트(Projects)로 작업 공간 분리하기

클로드의 프로젝트는 각 주제별로 독립된 메모리 공간입니다. 프로젝트마다 지침, 파일, 대화 기록이 따로 관리됩니다.

💡

왜 쓰나요? 프로젝트 없이 쓰면 우규 역삼점 리뷰 분석하다가 갑자기 다른 클라이언트 채널 정비를 하면 클로드가 맥락을 섞어버립니다. 프로젝트로 분리하면 각각 전문적으로 처리됩니다.

처음부터 만들어두면 좋은 프로젝트 예시

프로젝트 1: 클라이언트 관리실

이 프로젝트는 현재 활성 클라이언트 컨설팅 전용 공간입니다. - 항상 점포명·주소를 먼저 명시 (지점 혼동 절대 금지) - 체인점 여부 → 운영 권한 범위 먼저 확인 후 문서 설계 - 출력 가능한 A4 형식, 체크박스·테이블 우선 - 모든 문서는 점주 자립 운영 가능한 구조로 마무리 - 사업신청자 ≠ 실제 점주인 경우 반드시 분리 명시

업로드할 파일: 사전 질문지 응답, 리뷰 수집 데이터, 클라이언트 기본 정보

프로젝트 2: 컨설팅 자료 표준화

범용 컨설팅 프레임워크 및 템플릿 개발 전용입니다. - 모든 업종에서 바로 적용 가능한 형식으로 작성 - 사전 질문지 / 분석 기준표 / 포지셔닝 시트 / 채널 정비 체크리스트 - 업종별 특화 포인트는 별도 섹션으로 분리 - 버전 관리: 수정 시 날짜 기재

프로젝트 3: 상권·리뷰 분석

리뷰 분석 및 상권 경쟁 조사 전용입니다. - 분석 대상 점포명·주소를 항상 먼저 명시 - 네이버 플레이스·구글 리뷰·카카오맵 리뷰 종합 - 경쟁사 비교는 반드시 동일 상권 기준으로 - 인사이트는 4가지로 분류: 강점 언급 / 불만 언급 / 방문 목적 / 감성 언어

프로젝트 4: 퍼스널 브랜딩

권용준 컨설턴트 개인 브랜드 작업 전용입니다. - 인스타그램: @yjkwon_gangnamconsulting - 홈페이지: https://pyutyubeu88-commits.github.io/-/ - 포지셔닝: "강남구 소상공인을 AI로 직접 바꾸는 현장 컨설턴트" - 콘텐츠는 실제 컨설팅 사례 기반으로 작성

Claude Chat 실전 프롬프트 예시

포지셔닝 한 문장 설계가 필요할 때

클라이언트 매장의 포지셔닝 전략가 역할을 해줘. 상황을 설명하면 먼저 나에게 3가지 질문을 해줘. 그 다음에 선택지 나열 말고 실제 추천 방향을 알려줘. 매장 정보: - 이자카야 우규 역삼점 (서울 강남구 테헤란로19길 19) - 체인점 (지점 자체 수정 가능: 소개 문구·사진·영업시간) - 리뷰 강점 키워드: 넉넉한 양, 직원 친절, 회식 분위기 - 상권: 역삼 오피스 밀집 지역, 30~40대 직장인 중심

업종별 사전 질문지 생성

아래 업종에 맞는 사전 질문지를 작성해줘. 형식: A4 출력 기준, 체크박스와 기재란 포함. 포함 항목: 기본 정보 / 온라인 채널 현황 / 핵심 상품·서비스 / 사진 자산 / 고객 현황 / 점주 핵심 고민. 업종: F&B (이자카야) 추가 확인: 배달앱 등록 여부, 체인점 마케팅 제약 여부, 본사 채널 관리 여부

리뷰 인사이트 분석

아래 리뷰들을 분석해서 컨설팅 인사이트를 뽑아줘. 분류 기준: ① 강점 키워드 (반복 칭찬받는 것 3~5개) ② 불만 키워드 (반복 지적되는 것) ③ 방문 목적 (주로 어떤 이유로 오는가: 회식/데이트/일상 등) ④ 감성 언어 (리뷰에서 쓰는 분위기·느낌 표현) ⑤ 경쟁 비교 언급 ("다른 곳과 달리~" 류) [리뷰 붙여넣기]

Claude Cowork 완전 가이드

Cowork는 채팅이 아닙니다. "이 클라이언트 문서 만들어줘"라고 말하고 자리를 비우면, 돌아왔을 때 완성된 파일이 있습니다.

💰

플랜 안내 (변경될 수 있음)

  • Max ($100~$200/월): 더 많은 사용량
  • Cowork는 일반 채팅보다 토큰 소모가 5~20배 많습니다. 한도에 자주 도달한다면 Max 플랜을 고려하세요.
  • 무료 플랜은 Cowork 사용 불가

접근 방법: Claude Desktop 앱 다운로드(claude.ai/download) → Cowork 탭 클릭

3부

MCP 커넥터: 내가 쓰는 앱에 클로드 연결하기

MCP(Model Context Protocol)는 클로드를 내가 실제로 쓰는 앱에 연결하는 기술입니다. 연결하면 클로드가 그 앱의 데이터를 직접 읽고 액션을 취할 수 있습니다.

연결 가능한 서비스

  • Gmail, Google Drive, Google Calendar
  • Slack, Notion, GitHub
  • n8n (내가 만든 자동화 흐름을 클로드가 트리거)
  • Linear, Jira, Asana
  • 그 외 100개 이상

연결 방법: Settings → Connectors → 원하는 서비스 찾아서 연결

MCP 활용 실전 예시

Gmail 연동 — 클라이언트 문의 정리

오늘 받은 이메일 중 클라이언트 문의, 컨설팅 신청 관련된 것만 찾아줘. 각각 한 줄로 요약하고, 답장이 필요한 것은 초안 작성해줘. 내일 미팅 전에 확인해야 할 것도 알려줘.

Google Drive 연동 — 클라이언트 자료 찾기

Drive에서 '우규역삼점' 관련 파일 찾아줘. 사전 질문지 응답, 리뷰 분석 중 최신 버전을 찾아서 오늘 1회차 컨설팅에서 다룰 순서대로 정리해줘.

캘린더 연동 — 컨설팅 일정 관리

이번 주 컨설팅 일정 보여줘. 각 미팅 하루 전에 30분 준비 시간 잡아주고 금요일 오후에 주간 컨설팅 회고 1시간 넣어줘.
⚠️

MCP 커넥터는 필요할 때만 켜세요. 항상 켜두면 토큰 소모가 빠릅니다.

4부

스킬 vs 플러그인: 차이를 알면 활용도가 2배

스킬(Skill) — Claude Code & Cowork 전용

스킬 = 클로드에게 특정 작업 방법을 가르치는 마크다운 파일입니다. 한 번 만들어두면 관련 상황에서 자동으로 발동됩니다.

🎯

비유: 신규 컨설턴트에게 "우리 리뷰 분석은 이 4가지 기준으로 해"라고 매뉴얼을 써주는 것. 한 번 주면 매번 설명하지 않아도 됩니다.

스킬 만드는 법

  1. 폴더 생성: .claude/skills/positioning-design/
  2. 그 안에 SKILL.md 파일 생성:
# 포지셔닝 한 문장 설계기 ## 목적 클라이언트 매장 정보를 받으면 선택 이유가 담긴 포지셔닝 문장을 설계한다. ## 입력 정보 - 점포명·주소 (지점 혼동 금지) - 리뷰 강점 키워드 3~5개 - 주요 고객층과 방문 목적 - 경쟁사와 다른 점 ## 설계 규칙 - 실제 리뷰 언어에서 추출한 표현 사용 - 지어낸 강점 아닌 이미 있는 강점을 온라인에서 보이게 - 40자 이내의 한 문장 - 추상적 형용사 단독 사용 금지 ("맛있는", "친절한" 혼자 쓰면 X) ## 출력 형식 1. 강점 3가지 추출 (리뷰 기반) 2. 주요 고객의 선택 이유 (리뷰 언어 그대로) 3. 경쟁사에는 없는 것 4. 포지셔닝 문장 안 A / 안 B / 안 C 5. 적용 위치: 네이버 플레이스 소개 / 구글 비즈니스 / 인스타 바이오

만들어두면 유용한 스킬들

스킬명기능
pre-questionnaire업종·점포 정보 → 사전 질문지 자동 생성 (체크박스·기재란 포함)
review-analysis리뷰 붙여넣기 → 강점·불만·방문목적·감성언어 4가지 분류
positioning-design매장 정보 → 포지셔닝 한 문장 3가지 안 설계
channel-check채널 현황 → 정비 우선순위 체크리스트 (즉시 가능/승인 필요 분류)
consulting-homework컨설팅 내용 → 점주 숙제 목록 (난이도·기한 포함)

플러그인(Plugin) — Cowork 전용

플러그인 = 스킬 + 커맨드 + 커넥터가 하나로 묶인 패키지입니다. 특정 업무 분야를 위해 한 번 설치하면 그 분야 전문가처럼 동작합니다.

🎯

비유: 앱스토어에서 앱 하나 설치하면 그 기능을 바로 쓸 수 있는 것과 같습니다.

설치 방법: Cowork → Customize 메뉴 → 플러그인 찾기 → 설치

플러그인컨설팅에서 이럴 때 써라
Productivity클라이언트 일정 관리, 주간 브리핑 자동화
Marketing포지셔닝 문구 작성, 채널 정비 콘텐츠 기획

설치 후: Cowork 작업창에서 / 입력하면 설치된 플러그인 커맨드 목록이 뜹니다.

컨설팅에서 언제 쓰나요:

Extended Thinking

Search and tools → Extended thinking 토글 켜기. 클로드가 답하기 전에 먼저 깊이 생각하는 모드입니다.

이 매장의 포지셔닝 방향을 결정해야 해. 현재 리뷰에서 '양 많음'과 '직원 친절'이 반복되고 있어. '가성비 회식 이자카야' vs '분위기 있는 저녁 식사' 중 역삼 오피스 상권에서 어떤 방향이 더 유효할지 단기(6개월)·장기(2년) 관점 모두 고려해서 추천해줘.

더 깊이 생각하게 만드는 트리거 문장:

  • "답하기 전에 충분히 생각해줘"
  • "이 방향이 잘못됐을 경우를 먼저 생각해봐줘"
  • "6개월 후, 1년 후 어떤 영향이 있을지도 고려해줘"

과거 대화 검색 (유료 플랜)

자연어로 그냥 물어보면 됩니다.

  • "저번에 우규 역삼점 포지셔닝 어떻게 정했었지?"
  • "지난달에 작성했던 채널 정비 체크리스트 찾아줘"
  • "뷰티샵 사전 질문지 작업했던 대화 찾아줘"

스타일 설정

Settings → Style 에서 내 컨설팅 문서 샘플을 붙여넣으면 클로드가 내 문서 스타일을 학습해서 모든 작성에 적용합니다.

📝

실제로 현장에서 전달한 컨설팅 문서 3~5개를 붙여넣고 "이게 내 컨설팅 문서 스타일이야. 앞으로 문서 작성할 때 이 스타일 유지해줘"라고 하면 됩니다.

흔한 AI 디자인 패턴은 피해줘. 보라색 그라디언트, 평범한 시스템 폰트, 판에 박힌 카드 레이아웃 금지. 이 프로젝트만의 개성 있는 스타일로 만들어줘. 하나의 폰트를 자신 있게 쓰는 게 여러 폰트 어정쩡하게 쓰는 것보다 낫다.

주말 이틀 완전 설정 플랜

Day 1 — 기반 구축 (4시간)

1시간: Claude Chat 기본 설정

  • Settings → Capabilities에서 메모리 토글 켜기
  • Settings → Profile에서 컨설턴트 지침 작성 (위의 템플릿 사용)
  • 프로젝트 4개 만들고 각 지침 작성: 클라이언트 관리실 / 컨설팅 자료 표준화 / 상권·리뷰 분석 / 퍼스널 브랜딩
  • 각 프로젝트에 관련 파일 업로드 (사전 질문지, 리뷰 데이터 등)
  • Settings → Connectors에서 Gmail, Drive, 캘린더 연결

2시간: Cowork 설정

  • Claude Desktop 앱 설치 (claude.ai/download)
  • Cowork 탭 열고 /Clients 폴더 접근 테스트
  • 플러그인 2~3개 설치 (Cowork → Customize)
  • 첫 번째 예약 작업 만들기: 월요일 클라이언트 브리핑 (/schedule)

3시간: 스킬 파일 작성

  • .claude/skills/review-analysis/SKILL.md — 리뷰 분석 기준 정의
  • .claude/skills/positioning-design/SKILL.md — 포지셔닝 설계 규칙 정의
  • .claude/skills/consulting-homework/SKILL.md — 숙제 목록 형식 정의

4시간: Claude Code 설정 (자동화에 관심 있는 경우)

  • npm install -g @anthropic-ai/claude-code 설치
  • 컨설팅 작업 폴더에서 /init 실행
  • 생성된 CLAUDE.md에 현재 활성 클라이언트 정보 채우기
  • 간단한 파일 정리 작업으로 테스트

Day 2 — 실전 적용 (3시간)

1시간: 고급 기능 직접 써보기

  • Deep Research ON으로 현재 클라이언트 상권 조사 1건 실행
  • Extended Thinking ON으로 포지셔닝 방향 결정 1건 분석
  • 일반 채팅과 결과물 퀄리티를 직접 비교해보세요

2시간: 첫 번째 자동화 완성

  • 매주 가장 귀찮게 하는 반복 작업 1개 선택 (예: 리뷰 분류, 숙제 목록 정리)
  • Cowork 예약 작업으로 만들고 한 번 실행해서 결과 확인

3시간: 전체 점검

  • "나에 대해 뭘 기억하고 있어?" 라고 물어서 클로드 메모리 확인
  • 컨설턴트 지침 보완
  • 만든 스킬 테스트하고 수정
  • Claude Design으로 클라이언트 원페이저 1장 완성해보기

처음 시작할 때 쓰는 마스터 프롬프트

내 컨설팅 작업에 맞는 클로드 설정을 처음부터 제대로 구축하고 싶어. 나에 대해: - 직업/역할: 강남구 AI 마케팅 자립 컨설턴트 - 주요 클라이언트: 강남구 로컬 비즈니스 점주 (F&B·뷰티·의료·교육·피트니스) - 주요 사용 목적: 사전 질문지 / 리뷰 분석 / 포지셔닝 설계 / 채널 정비 / 숙제 목록 매주 반복되는 귀찮은 작업 3가지: 1. 새 클라이언트마다 사전 질문지를 처음부터 다시 작성 2. 리뷰 수동으로 읽고 강점·불만·방문목적으로 분류하기 3. 컨설팅 후 숙제 목록 정리하고 난이도·기한 붙여서 문서화하기 내가 현재 쓰는 도구: Gmail, Google Drive, Google Calendar, 네이버 플레이스, 구글 비즈니스 아래 내용으로 내 상황에 딱 맞는 설정 계획을 만들어줘: 1. 만들어야 할 프로젝트 목록과 각 지침 내용 2. 먼저 만들어야 할 스킬 파일 3. 설치할 Cowork 플러그인 4. 우선 연결할 MCP 커넥터 5. 제일 먼저 만들 예약 작업 6. 자동화할 것 vs 수동으로 남길 것 카테고리 말고, 실제로 뭘 써야 하는지 구체적으로 알려줘.

설치 방법

터미널(Mac: 터미널 앱, Windows: PowerShell)에서 아래를 입력하세요:

npm install -g @anthropic-ai/claude-code

설치 후 컨설팅 작업 폴더로 이동해서:

cd 내-컨설팅-폴더 claude

처음 실행하면 브라우저로 로그인 화면이 뜹니다. 로그인하면 바로 사용 가능합니다.
