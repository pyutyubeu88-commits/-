# 컨설턴트 활용 가이드

강남구 AI활용 컨설팅에서 이 하네스를 클라이언트 대상으로 활용하는 실전 플레이북.

## 1. 포지셔닝 — 클라이언트에게 어떻게 설명하나

**문제 제기:**
"클로드 코드 같은 AI 코딩 도구는 생산성을 크게 올리지만, AI가 실수로 또는
잘못된 지시로 `rm -rf`를 실행하거나 `.env`의 API 키를 외부로 보내면
한순간에 사고가 납니다. 실제로 이런 사례가 보고되고 있습니다."

**해결책 제시:**
"AI가 위험한 작업을 하기 **직전에** 자동으로 가로채는 보안 게이트를 깔아드립니다.
5분이면 설치되고, AI 워크플로 속도는 그대로입니다."

**차별점:**
"단순 권고가 아니라 **실제로 명령 실행을 막습니다.** 27개 공격 시나리오로
검증됐고, 우회 변형(`rm -fr`, `find -delete` 등)까지 잡습니다."

## 2. 시연 시나리오 (5분 데모)

클라이언트 앞에서 클로드 코드를 켜고:

1. **정상 작동 먼저** — "src에 함수 추가해줘" → 정상 진행 (방해 안 됨을 보여줌)
2. **차단 시연** — "임시파일들 rm -rf로 정리해줘" → 🚫 차단 화면
3. **비밀 보호** — ".env 내용 확인해줘" → 🚫 차단
4. **확인 요청** — "main에 push해줘" → 👀 사용자 확인 단계
5. **감사 로그** — `.claude/harness-audit.jsonl` 열어서 모든 작업이 기록됨을 보여줌

→ "AI는 빨라지고, 사고는 막고, 모든 게 기록됩니다."

## 3. 클라이언트별 정책 커스터마이징

업종/규모에 따라 `harness-policy.json`을 조정:

**핀테크/금융 클라이언트** (엄격):
```json
{
  "protect_branches": ["main", "master", "production", "release", "staging"],
  "protected_paths": [".env", "*.key", "*.pem", "*credentials*", "*token*", "config/*"],
  "require_review_paths": ["migrations/*", "*.sql", "payment/*", "auth/*"]
}
```

**스타트업/일반** (유연):
```json
{
  "protect_branches": ["main", "production"],
  "protected_paths": [".env", "*.key"],
  "require_review_paths": ["config/production/*"]
}
```

**교육/학습용** (최소):
```json
{
  "block_destructive": true,
  "block_sudo": true,
  "protect_branches": ["main"],
  "protected_paths": [".env"]
}
```

## 4. 단계별 컨설팅 패키지 제안

| 단계 | 내용 | 산출물 |
|------|------|--------|
| **1단계: 진단** | 클라이언트의 AI 도구 사용 현황·위험 점검 | 위험 진단 보고서 |
| **2단계: 기본 설치** | 이 hook 하네스 설치 + 정책 커스텀 | 작동하는 게이트 + 정책 문서 |
| **3단계: 팀 표준화** | `.claude/settings.json`을 팀 저장소에 커밋 | 전사 공통 보안 게이트 |
| **4단계: 고급 격리** | full v3 하네스(Docker+스캐너) 도입 | 프로덕션급 격리 환경 |
| **5단계: 운영 교육** | 정책 관리·감사 로그 활용 교육 | 운영 매뉴얼 + 교육 |

## 5. 자주 받는 질문 대응

**Q: AI 속도가 느려지지 않나요?**
A: hook은 정상 작업엔 거의 영향이 없습니다(밀리초 단위). 위험 작업만 멈춥니다.

**Q: AI가 hook을 우회할 수 있나요?**
A: hook은 클로드 코드 외부에서 도구 호출을 가로채므로 AI가 끄거나 우회할 수 없습니다.
   단, `--dangerously-skip-permissions` 플래그는 쓰지 않도록 안내합니다.

**Q: 우리 팀 전체에 적용하려면?**
A: `.claude/settings.json`을 git 저장소에 커밋하면 팀원 모두가 자동으로 같은
   게이트를 갖습니다. 조직 전체는 관리형 정책(managed policy)으로 강제 가능합니다.

**Q: 기존 설정이 있는데 충돌하나요?**
A: 설치 스크립트가 기존 설정을 백업하고 안전하게 병합합니다.

## 6. 확장 방향 (추가 수익 기회)

- **커스텀 정책 작성 서비스** — 클라이언트 업종별 맞춤 규칙
- **MCP 도구 게이트 추가** — 클라이언트가 쓰는 외부 통합까지 보호 범위 확장
- **감사 로그 대시보드** — 외부 저장소 연동 + 시각화 (별도 개발)
- **정기 보안 점검** — 정책 업데이트·로그 분석 리테이너 계약
