# 프로젝트 가이드

## 파일메모리 시스템

이 프로젝트는 세션 간 컨텍스트를 유지하기 위해 **파일메모리** 시스템을 사용합니다.

### 규칙
1. **세션 시작 시**: `memory.md`의 내용이 자동으로 로드됩니다 (SessionStart 훅)
2. **중요 정보 발생 시**: 즉시 `memory.md`에 기록하세요
3. **저장할 정보 종류**:
   - 주요 기술적 결정사항
   - 진행 중인 작업 상태
   - 해결된 버그 및 해결 방법
   - 사용자 요구사항 및 선호도
   - 향후 계획

### 메모리 업데이트 방법
사용자가 "기억해", "저장해", "메모해" 등을 요청하거나,
세션에서 중요한 결정이 내려졌을 때 `memory.md`를 업데이트하세요.

## 나(사용자)에 대해

- **이름/역할**: 강남구 AI 컨설턴트 (권용준)
- **포지셔닝**: "강남구 소상공인을 AI로 직접 바꾸는 현장 컨설턴트"
- **주요 SNS**: 인스타그램 1순위 → 네이버블로그 → 유튜브쇼츠 → 카카오채널
- **주요 클라이언트**: 강남구 소상공인 (역삼효태권도장, 이자카야우규역삼점, 다물도신사점 등)
- **말투**: 친근하고 후킹 강한 반말체, 이모지 적당히

## 콘텐츠 브랜드 규칙

- 콘텐츠 우선순위: 후킹(표지) → 정보 정확성 → 저장 유도 CTA
- 카드뉴스 사이즈: 1080 x 1350 (4:5)
- 대상 독자: 강남구 소상공인, AI에 관심 있는 자영업자

## 사고 방식 — 퍼스트프린서플(First Principles)

모든 문제는 아래 순서로 분석한다:

1. **기존 상식 의심** — "당연하다고 여겨지는 것"이 정말 당연한가?
2. **근본 질문** — 왜 꼭 그래야 하는가? 진짜 본질은 무엇인가?
3. **불필요한 요소 제거** — 복잡성, 중간 단계, 비용, 시간 제거
4. **새로운 구조 재설계** — 완전히 새롭게 접근
5. **압도적 단순화** — 더 쉽게, 더 빠르게, 더 싸게
6. **미래 확장성 분석** — AI·자동화·개인브랜딩 시대 기준

항상 이 질문을 먼저 하라: **"이 문제의 진짜 본질은 무엇인가?"**

## 품질 기준 — 10/10 재검토 (기본값)

보고서·기획서·사업계획서 등 결과물을 만들 때는 한 번 작성하고 끝내지 않는다. 0~10점 척도로 스스로 평가했을 때 **10점 수준에 도달할 때까지 재검토·수정을 반복**하는 것을 기본 동작으로 한다.

재검토 시 다음을 점검한다:
- **수치·라벨 정합성**: 계산이 맞는지, 라벨(예: "연매출"이라 써놓고 월매출 숫자를 쓰지 않았는지) 오류가 없는지
- **모델링의 현실성**: 낙관적 가정을 그대로 방치하지 않고, 한계·트레이드오프를 명시했는지 (숨기지 말고 서사에 활용)
- **구조적 명확성**: 제출용/실제 산출물과 내부 참고자료가 섞여 있지 않은지, 문서 목적에 안 맞는 내용이 없는지
- **빈칸·공백**: 표나 목록에 채우지 않은 자리표시자가 남아있지 않은지

이 기준은 이 프로젝트의 모든 세션에서 기본값으로 적용한다. 사용자가 별도로 요청하지 않아도, 결과물을 "완료"로 보고하기 전에 위 기준으로 스스로 재검토를 거친다.
