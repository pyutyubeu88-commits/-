# 프로젝트 가이드

## 파일메모리 시스템

이 프로젝트는 세션 간 컨텍스트를 유지하기 위해 **파일메모리** 시스템을 사용합니다.

### 규칙
1. **세션 시작 시**: `memory.md`의 내용이 자동으로 로드됩니다 (SessionStart 훅)
2. **중요 정보 발생 시**: 즉시 `memory.md`에 기록하세요
3. **저장할 정보 종류**:
   - 주요 기술적 결정사항
   - 진행 중인 작업 상태
   - 해결된 버그 및 해결 방법
   - 사용자 요구사항 및 선호도
   - 향후 계획

### 메모리 업데이트 방법
사용자가 "기억해", "저장해", "메모해" 등을 요청하거나,
세션에서 중요한 결정이 내려졌을 때 `memory.md`를 업데이트하세요.

## 프로젝트 정보

- **프로젝트**: AI 자동 대댓글 생성기
- **기술 스택**: 순수 HTML/CSS/JavaScript (단일 파일)
- **언어**: 한국어 UI

## 에이전트 시스템 (서브 에이전트 & 에이전트 팀)

복잡하거나 규모가 큰 작업은 여러 에이전트에 나눠 처리합니다.

### 선택 기준
- **작업이 독립적인가?**
  - YES → **서브 에이전트** (병렬 처리, 비용 절감, 컨텍스트 격리)
  - NO (팀원 간 정보 공유 필요)
    - 팀원마다 다른 스킬 세트 필요 → **서브 에이전트** (선택적 스킬 주입)
    - 같은 스킬 세트 공유 가능 → **에이전트 팀** (협업·소통)

### 프로젝트 전용 서브 에이전트 (`.claude/agents/`)
- **frontend-reviewer**: HTML/CSS/JS 단일 파일 코드 리뷰 (읽기 전용)
- **security-auditor**: API 키·OAuth 토큰 노출 등 클라이언트 보안 점검 (읽기 전용)
- **landing-copywriter**: 강남구 AI 컨설팅 랜딩페이지 한국어 카피 작성

### 비용 규칙 (매 세션 자동 적용)
1. **모델 믹싱**: 리드는 Opus(분배·계획 검토·품질 판단), 팀원·서브 에이전트는 **Sonnet**(코드 생성·편집). Opus는 Sonnet 대비 토큰 4~5배 소비.
2. **팀 필요성 판단**: 팀원 간 소통이 필요 없으면 에이전트 팀 대신 **서브 에이전트** 사용.
3. **최소 인원 유지**: 대부분 작업은 **3~5명**으로 충분. 인원이 많을수록 조율 오버헤드만 증가.
4. **작업 완료 즉시 종료**: 끝난 팀원은 리드에게 종료 요청, 전체 완료 후 "팀을 정리해줘"로 일괄 종료(살아있는 세션은 토큰 계속 소모).
5. **결과 검증**: 에이전트 결과는 검증 후 사용. 의존성 있는 작업을 무리하게 병렬화하지 않음.

### 계획 승인 (Plan Approval)
팀원이 코드를 수정하기 전 계획을 제출·승인받게 하려면 팀 생성 프롬프트에 추가:
> "각 팀원이 변경하기 전에 계획 승인을 받도록 해줘."

자동 차단/승인 기준 예: DB 스키마 수정 계획은 자동 차단, 테스트 커버리지 포함 계획만 승인.
