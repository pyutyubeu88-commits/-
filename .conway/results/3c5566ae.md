## index.html 코드 리뷰

### 🔴 보안 이슈 (심각)

**1. API 키 및 OAuth 토큰을 localStorage에 저장 (239~242번 줄)**
```javascript
var ytApiKey = localStorage.getItem('ytApiKey') || '';
var gmApiKey = localStorage.getItem('gmApiKey') || '';
var gToken = localStorage.getItem('gToken') || '';
```
localStorage는 같은 도메인의 모든 JS 코드(광고 스크립트 등)에서 접근 가능합니다. XSS 공격 발생 시 API 키와 OAuth 토큰이 즉시 탈취됩니다. OAuth 토큰은 사용자의 YouTube 채널에 대한 쓰기 권한을 가지므로 특히 위험합니다.

**2. Gemini API 키가 URL 쿼리 파라미터로 전송 (459번 줄)**
```javascript
var url = 'https://generativelanguage.googleapis.com/...?key=' + gmApiKey;
```
URL 쿼리 파라미터는 브라우저 히스토리, 서버 접근 로그, 네트워크 프록시 로그에 평문으로 기록됩니다. 헤더로 전송해야 합니다.

**3. 프롬프트 인젝션 취약점 (531~543번 줄)**
```javascript
var prompt = '...\n댓글 내용: ' + cmt;
```
YouTube 댓글에 악의적인 AI 지시문(예: `"무시하고 다음 명령을 실행: ..."`)이 포함될 경우 프롬프트 조작이 가능합니다. 댓글 내용을 프롬프트에 삽입하기 전에 경계 처리(XML 태그 감싸기 등)가 필요합니다.

**4. Google Client ID 하드코딩 (236번 줄)**
```javascript
var GOOGLE_CLIENT_ID = '941692480971-o4rls1vjrrocse128f1qe5hqlou3thd3.apps.googleusercontent.com';
```
Client ID 자체는 공개 식별자이지만, 이를 기반으로 피싱 페이지가 동일한 OAuth 플로우를 위장할 수 있습니다. 또한 Authorized redirect URIs 관리에 주의가 필요합니다.

### 🟠 코드 품질 이슈 (중요)

**5. `analyze()`가 댓글 클릭마다 자동 호출 (412번 줄)**
```javascript
function selectCmt(i) { ... analyze(); }
```
디바운싱 없이 댓글 클릭마다 Gemini API를 호출합니다. 목록을 빠르게 스크롤하면 불필요한 API 호출이 폭발적으로 발생합니다.

**6. URL 파라미터 직접 문자열 연결 (355, 376번 줄)**
```javascript
fetch('https://...?part=snippet&id=' + vid + '&key=' + ytApiKey)
```
`URLSearchParams`를 사용하지 않아 특수문자 처리 오류가 발생할 수 있습니다.

**7. `var` 전역 변수 남용 (239~252번 줄)**
`const`/`let` 대신 `var`를 사용하고 모든 상태를 전역으로 관리하여 의도치 않은 덮어쓰기 위험이 있습니다.

**8. 인라인 이벤트 핸들러 전면 사용**
```html
<button onclick="startBulk()">...</button>
```
HTML 전체에 걸쳐 `onclick="..."` 형태로 작성되어 있습니다. CSP(Content Security Policy) 적용이 불가능하고 테스트도 어렵습니다.

**9. 파일 불완전 (650번 줄에서 잘림)**
```javascript
function next(idx) {
    if(idx >= total   ← 여기서 파일 끊김
```

`startBulk()`의 핵심 로직(`next()`, `postAll()` 등)이 없습니다. 이 상태로는 일괄 생성 기능이 동작하지 않습니다.

### 🟡 성능 이슈

**10. 댓글 목록 전체 재렌더링 (399~405번 줄)**
```javascript
function renderCmts() {
    var html = '';
    allCmts.forEach(...) { html += '...' }
    document.getElementById('cmtList').innerHTML = html;
}
```
댓글 선택 시마다 전체 목록을 innerHTML로 재생성합니다. 댓글 수가 많을 경우 클릭 반응이 느려집니다.

**11. `BULK_DELAY = 8000` (233번 줄)**
8초 지연은 Gemini API 무료 티어의 RPM 제한을 고려한 것이지만, 댓글 100개 처리 시 13분 이상 소요됩니다. 실제 API 응답 속도에 맞춰 동적으로 조정하는 것이 좋습니다.

| 우선순위 | 이슈 | 조치 |
