# index.html 코드 리뷰 결과

## ✅ 잘 된 점
- CSS 변수(`:root`)로 디자인 시스템 통일 → 유지보수 용이
- `esc()` 함수로 기본 XSS 방어 구현됨
- localStorage로 API 키 영속 저장
- `safeJSON()`으로 JSON 파싱 오류 방어

## 🔴 보안 이슈 (우선 처리 필요)

### 1. API 키 평문 노출
```javascript
var GOOGLE_CLIENT_ID='941692480971-o4rls1vjrrocse128f1qe5hqlou3thd3.apps.googleusercontent.com';
```
- **문제**: Google Client ID가 소스코드에 하드코딩됨
- **위험도**: 중간 (Client ID 자체는 공개 가능하나 환경 분리 권장)

### 2. Access Token localStorage 저장
```javascript
localStorage.setItem('gToken', gToken);
```
- **문제**: OAuth Access Token을 localStorage에 저장 → XSS 공격 시 탈취 가능
- **권장**: 메모리(변수)에만 보관, 세션 만료 시 재인증

### 3. `textContent` 미사용 구간
```javascript
document.getElementById('vidTitle').textContent = s.title;  // ✅ 안전
document.getElementById('anType').textContent = a.type;     // ✅ 안전
```
대부분 안전하나 동적 HTML 삽입 구간 재확인 필요

## 🟡 성능 이슈

### 4. BULK_DELAY 8초 하드코딩
```javascript
var BULK_DELAY = 8000;
```
- YouTube API 할당량 보호를 위한 것이나 사용자 설정 불가

### 5. 댓글 100개 고정
```javascript
commentThreads?...&maxResults=100
```
- 페이지네이션 미구현 → 댓글 많은 영상에서 누락

### 6. 전역 변수 남용
`allCmts`, `selIdx`, `bulkResults` 등 전역 상태가 많아 디버깅 어려움

## 🟢 개선 제안

### 7. API 키 입력 UX
현재 매 세션마다 새로 저장해야 함 → 저장 즉시 도트 색상 변경은 좋으나,
페이지 로드 시 마스킹된 형태로 표시 (`AIza****xyz`) 추천

### 8. 오류 처리 강화
```javascript
.catch(function(e){ toast('오류: ' + e.message); })
```
Gemini API 할당량 초과(429), 잘못된 키(400) 등 상태 코드별 안내 메시지 추가 권장

### 9. 일괄 생성 중단 기능 없음
`startBulk()` 실행 후 취소 불가 → 중단 버튼 추가 필요
